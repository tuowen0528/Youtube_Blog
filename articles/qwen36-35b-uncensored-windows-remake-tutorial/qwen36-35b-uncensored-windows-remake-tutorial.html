Qwen3.6 · llama.cpp · CUDA · Windows

旧版有坑！Qwen3.6 35B越狱版本地部署【重制教程】｜12GB显存实测

本期重新演示在 Windows 上使用 llama.cpp 部署 Qwen3.6-35B-A3B Uncensored 模型。测试电脑使用 RTX 3060 12GB， 选择 IQ2_M 量化版本，并补齐上期教程遗漏的 CUDA DLL 和参数配置。

发布于 2026 年 6 月 22 日 视频时长 6:08 Windows · NVIDIA CUDA
在 YouTube 观看本期视频
这篇文章替代旧版部署参数。 上一期教程中的启动脚本没有根据显存和并行 slot 做完整调优， 可能出现 CUDA 未加载、显存被挤满或生成速度很低的问题。 如果你已经按旧教程部署，建议保留模型文件，重新下载匹配版本的 llama.cpp 与 CUDA DLL， 再使用本文脚本。

开始前要准备什么

  • Windows 10 或 Windows 11 电脑，并能正常解压 ZIP 文件。
  • NVIDIA 显卡和可用的 NVIDIA 驱动。视频测试环境是 RTX 3060 12GB。
  • 至少约 13GB 的可用磁盘空间，用于 IQ2_M 主模型、视觉模型和 llama.cpp。
  • 下载大文件时保持网络稳定。Hugging Face 模型文件超过 10GB。
  • 部署前关闭占用大量显存的软件，尤其是其他 AI 程序、NVIDIA Broadcast、录屏和重度浏览器页面。
显存不同，模型选择也应不同。 本文的 IQ2_M 与启动参数是针对 12GB 显存测试的。 显存更大时可以考虑更高质量的量化；显存更小时，即使模型能启动，也可能频繁使用共享内存而变慢。

为什么要重做上一期教程

上一期已经能够把模型启动起来，但“能运行”不代表配置正确。 当 CUDA 运行库缺失时，脚本即使写了 GPU 层参数，llama.cpp 仍可能只使用 CPU； 默认创建多个服务 slot、上下文过大或显存贴边，也会让 CUDA 推理变得很慢。

本次重新测试后，12GB 显存环境采用 IQ2_M 主模型、F16 视觉投影模型、Q8 KV、 8192 上下文和单 slot。后文会给出完整脚本，也会链接到独立性能数据页面。

下载 Windows CUDA 主程序和运行库

  1. 向下找到 Windows x64 下载项

    展开当前版本的 Assets，向下找到 Windows 区域。 NVIDIA 显卡选择名称中带有 win-cuda 的 x64 压缩包。 视频选择 CUDA 13 版本，文件名通常类似： 

    llama-b9754-bin-win-cuda-13.3-x64.zip

    版本号可能已经更新，文件名前半部分会不同。AMD、Intel 或纯 CPU 用户不能照搬这个下载项。

  2. 同时下载匹配的 CUDA DLL

    只下载 llama.cpp 主压缩包还不够。继续找到同一版本下的 CUDA 13.3 DLLs，文件名通常类似： 

    cudart-llama-bin-win-cuda-13.3-x64.zip

    两个压缩包的 CUDA 大版本必须匹配。主程序使用 CUDA 13 时，不要混用 CUDA 12 的 DLL。

03:00

把 CUDA 13.3 DLL 复制到程序根目录

  1. 解压 CUDA DLL 压缩包

    找到另一个下载的 cudart-llama-bin-win-cuda-13.3-x64.zip 并解压。

  2. 选中三个 DLL

    进入解压后的目录，应看到以下文件：

    cublas64_13.dll
cublasLt64_13.dll
cudart64_13.dll

    选中三个文件后按 Ctrl + C。

  3. 粘贴到 llama-server.exe 同级目录

    回到 llama.cpp 根目录，也就是能看到 llama-server.exe 的位置，按 Ctrl + V 粘贴。

  4. 可选：先检查 CUDA 是否被识别

    在根目录打开 PowerShell，运行： 

    .\llama-server.exe --list-devices

    正常时应看到类似：

    CUDA0: NVIDIA GeForce RTX 3060

    如果只出现 Available devices: 而没有 CUDA0， 先不要继续跑模型。检查主程序与 DLL 的 CUDA 版本是否匹配，并更新 NVIDIA 驱动。

03:48

创建经过调优的 start.bat

视频把准备好的启动脚本拖入 llama.cpp 根目录。 你也可以右键新建文本文件，将文件名改为 start.bat， 再右键选择 编辑，写入以下完整内容。 

@echo off
chcp 65001 >nul
cd /d "%~dp0"

llama-server.exe ^
-m "models\Qwen3.6-35B-A3B-Uncensored-HauhauCS-Aggressive-IQ2_M.gguf" ^
--mmproj "models\mmproj-Qwen3.6-35B-A3B-Uncensored-HauhauCS-Aggressive-f16.gguf" ^
--mmproj-offload ^
--image-min-tokens 1024 ^
--image-max-tokens 1024 ^
-ngl all ^
--fit off ^
--flash-attn on ^
--cache-type-k q8_0 ^
--cache-type-v q8_0 ^
-c 8192 ^
-n 4096 ^
-np 1 ^
--host 127.0.0.1 ^
--port 8080

pause

主要参数为什么这样设置

  • --mmproj-offload：把视觉投影计算卸载到 GPU。
  • --image-min-tokens 1024 与 --image-max-tokens 1024：减少普通图片首轮处理时间。
  • -ngl all --fit off：尽可能把主模型层放到 GPU，并关闭自动显存适配。
  • --cache-type-k q8_0 与 --cache-type-v q8_0：本机实测比 Q4 KV 更快。
  • -c 8192：在 12GB 显存环境保留实用上下文长度。
  • -np 1：只创建一个服务 slot，避免默认多个 slot 增加 KV 与显存压力。
  • --host 127.0.0.1：只允许当前电脑访问，不直接暴露到局域网。
脚本中的文件名必须与本机完全一致。 如果你下载了其他量化版本、给文件改过名，或 llama.cpp 更新后参数发生变化， 应同步修改脚本。命令窗口提示“failed to open model”时，首先检查路径和文件名。

常见问题和故障排查

生成速度仍然只有约 10–15 t/s

  • 运行 .\llama-server.exe --list-devices，确认存在 CUDA0。
  • 检查三个 CUDA 13 DLL 是否与 llama-server.exe 位于同一目录。
  • 确认 DLL 与下载的 llama.cpp CUDA 大版本一致。
  • 查看任务管理器或 nvidia-smi，关闭其他占用显存的程序。
  • 检查脚本是否保留 -np 1，避免默认多个 slot 挤占显存。
  • 首次请求可能包含 CUDA 预热，连续测试两三次后再记录稳定速度。

启动时显存不足或程序退出

  • 先关闭所有其他 GPU 应用并重新启动脚本。
  • 把 -ngl all --fit off 改为 -ngl auto -fitt 256。
  • 仍然失败时降低上下文，例如把 -c 8192 改为 -c 4096。
  • 确认下载的是 IQ2_M，而不是体积更大的 IQ3_M。

模型能聊天，但不能识别图片

  • 确认 F16 mmproj 文件位于 models 文件夹。
  • 确认 --mmproj 后的文件名和本机文件完全一致。
  • 启动日志中不应出现 mmproj 文件无法打开的错误。
  • 换一张普通 JPG 或 PNG 测试，排除图片格式问题。

图片识别首轮等待很久

  • 确认脚本包含 --image-min-tokens 1024 和 --image-max-tokens 1024。
  • 1024 偏向速度，适合普通场景描述；小字 OCR、密集表格和精细定位可能需要更高值。
  • 报告中的 1920 × 1200 测试图从约 36.4 秒降到 11.65 秒，但不同图片会有差异。

双击 start.bat 一闪而过

  • 确认脚本末尾保留 pause，这样错误不会立即关闭。
  • 打开文件扩展名显示，检查是否误保存为 start.bat.txt。
  • 检查命令中的引号、续行符 ^ 和模型文件名。
